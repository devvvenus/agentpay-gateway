# Reference Behaviors

- Navigation remains fixed over the hero and collapses to brand plus one action on mobile.
- The WebGL model loops continuously, responds subtly to pointer tilt and changes transform with scroll.
- The product interface overlaps the hero boundary instead of occupying a separate framed section.
- Desktop feature rows alternate media and text; mobile stacks them vertically.
- Buttons use compact pill geometry with black/white inversion.
- Cards do not float through heavy shadows; hierarchy comes from scale, whitespace and dark/light section changes.
- Mobile uses one-column cards, smaller headings and a reduced product preview while retaining the same section order.
